Nuora daily COGS API for MobyBots

Use this endpoint to fetch daily cost of goods sold values for profit tracking. The API returns only COGS values and finality status, so MobyBots can combine it with its own Shopify revenue data.

usd cents zurich dates daily or range bearer token

Endpoint

Send a GET request to:

https://nuora-finance.vercel.app/api/integrations/cogs/daily

Authentication

Include the API key in the Authorization header:

Authorization: Bearer <api_key>
The API key is provided privately. It is not published on this page.

Input

The endpoint accepts an inclusive date range using Zurich calendar dates.

Parameter Required Format Meaning
startDate yes YYYY-MM-DD First day to return.
endDate yes YYYY-MM-DD Last day to return. Must be the same as or after startDate.
  • Maximum range: 90 days.
  • Historical dates are supported from 2026-01-14 onward.
  • Dates before 2026-01-14 are rejected because Nuora does not have complete raw Shopify/supplier source-of-truth data before that date.
  • Future dates are not supported.
  • Use the same date for startDate and endDate to fetch one day.

Example Requests

One day

curl -X GET "https://nuora-finance.vercel.app/api/integrations/cogs/daily?startDate=2026-06-05&endDate=2026-06-05" \
  -H "Authorization: Bearer <api_key>"

Date range

curl -X GET "https://nuora-finance.vercel.app/api/integrations/cogs/daily?startDate=2026-06-02&endDate=2026-06-06" \
  -H "Authorization: Bearer <api_key>"

Output

The response is JSON. COGS is returned in cents, in USD.

Field Type Meaning
schemaVersion string Response schema version.
timezone string Always Europe/Zurich.
currency string Always USD.
range object The requested startDate and endDate.
rangeStatus string final, estimated, or mixed.
generatedAt string ISO timestamp when the response was generated.
days array One object per date.
days[].date string Calendar date.
days[].cogsCents integer COGS in USD cents. Example: 3234916 means $32,349.16.
days[].status string final or estimated.

Status Logic

  • final: COGS for that day is booked/reconciled from supplier data.
  • estimated: COGS for that day is still projected because supplier cost data is not fully closed yet.
  • mixed: the requested range contains both final and estimated days.
For today and other open days, MobyBots should treat cogsCents as the current best COGS value and use the status field to label it as estimated.

Error Responses

Status Code Meaning
401 UNAUTHORIZED Missing or invalid bearer token.
403 FORBIDDEN The API key does not have cogs:read access.
400 INVALID_DATE_RANGE Bad date format or startDate after endDate.
400 UNSUPPORTED_DATE_RANGE Date is before Nuora's available source-data history or in the future.
400 DATE_RANGE_TOO_LARGE Range is over 90 days.
